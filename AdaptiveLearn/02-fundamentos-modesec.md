# 02. Fundamentos iniciales desde un enfoque basado en competencias

## 1. Punto de partida

Si trabajamos la documentación con una lógica basada en competencias, el proyecto no debe comenzar por una asignatura concreta ni por las funcionalidades técnicas, sino por la transformación formativa que se espera lograr en el estudiante.

## 2. Principio orientador

Primero se define **qué debe aprender el estudiante**, luego **cómo se evidenciará ese aprendizaje**, después **qué mediaciones necesita según su perfil**, y finalmente **qué funciones tecnológicas debe ofrecer el sistema para apoyar ese proceso**.

## 3. Base documental mínima

Antes de diseñar pantallas, módulos o flujos, conviene documentar estos elementos:

### a. Competencia objetivo
¿Qué será capaz de hacer el estudiante en un contexto significativo?

### b. Resultados de aprendizaje
¿Qué desempeños observables permitirán afirmar que avanza en la competencia?

### c. Evidencias
¿Qué productos, respuestas, interacciones o actividades servirán para evaluar el progreso?

### d. Criterios de evaluación
¿Cómo se distinguirá un desempeño inicial, intermedio o logrado?

### e. Estrategias de mediación
¿Qué tipo de recursos, apoyos, secuencias o formatos necesita cada perfil de estudiante?

### f. Retroalimentación
¿Qué tipo de feedback dará el sistema al estudiante y al docente?

## 4. Traducción de la necesidad pedagógica a requisitos del sistema

| Componente pedagógico | Traducción al software |
|---|---|
| Competencia | Motor de objetivos de aprendizaje |
| Resultado de aprendizaje | Indicadores de progreso |
| Evidencia | Actividades, entregables, trazas de interacción |
| Criterio de evaluación | Rúbricas, reglas, analítica |
| Mediación didáctica | Generación de recursos adaptativos |
| Retroalimentación | Paneles, recomendaciones y mensajes personalizados |

## 5. Cómo debe pensarse AdaptiveLearn desde este enfoque

AdaptiveLearn no solo debe generar contenido, sino sostener una lógica formativa completa:

- diagnosticar el punto de partida del estudiante;
- construir un perfil de aprendizaje útil para la intervención docente;
- vincular recursos a competencias y evidencias;
- adaptar la mediación sin perder el objetivo curricular;
- evaluar proceso y resultado;
- informar al docente para intervenir mejor.

## 6. Advertencia importante

Un error común en software educativo es empezar por “qué puede hacer la IA”. Un enfoque sólido empieza por:

- qué debe aprender el estudiante;
- cómo se demostrará ese aprendizaje;
- qué información necesita el docente para acompañarlo;
- qué parte de ese proceso conviene automatizar.

## 8. Relación entre MODESEC y AdaptiveLearn

Desde esta lógica, AdaptiveLearn debe documentarse y diseñarse como un sistema que:

- parte de competencias y no solo de contenidos;
- relaciona perfil del estudiante con estrategias de mediación;
- convierte evidencias de desempeño en decisiones adaptativas;
- ofrece apoyo tanto al estudiante como al docente;
- permite aplicar el mismo modelo en distintas áreas del currículo.
