# 05. Caso piloto inicial

## 1. Propósito del documento

Este documento define el primer caso piloto con el que se validará AdaptiveLearn en un contexto concreto. Su función es aterrizar la visión general del sistema en una implementación pequeña, medible y realista.

## 2. Aclaración importante

El caso piloto **no define todo el producto**. Solo representa el primer contexto donde se comprobará si el modelo de diagnóstico, personalización y generación de recursos funciona en la práctica.

## 3. Elementos que debe definir el piloto

- nivel educativo;
- área o asignatura inicial;
- tema o unidad específica;
- competencia a desarrollar;
- evidencias de aprendizaje;
- tipo de diagnóstico inicial;
- tipos de recurso a personalizar;
- métricas de validación.

## 4. Estado actual

Actualmente el proyecto ha avanzado hacia una definición más concreta del piloto y ha establecido como decisión orientadora:

- **Nivel educativo:** secundaria
- **Tipo de institución:** colegios tradicionales
- **Área priorizada para el piloto:** inglés
- **Tema priorizado:** Present Simple

Estas decisiones pueden ajustarse si la validación del proyecto lo requiere, pero por ahora representan la ruta piloto más coherente con la documentación construida.

## 5. Criterio para escoger un buen piloto

Un buen piloto debe:

- tener una competencia fácil de observar;
- permitir medir progreso en poco tiempo;
- facilitar la creación de recursos adaptativos;
- ser comprensible para docentes y estudiantes;
- producir evidencia útil para validar el sistema.

## 6. Preguntas pendientes del piloto

- ¿Qué grado de secundaria participará primero en la prueba?
- ¿Qué versión exacta del diagnóstico inicial tendrá el piloto?
- ¿Qué recursos se incluirán en la primera iteración y cuáles se dejarán para una segunda?
- ¿Qué duración tendrá el piloto?
- ¿Qué criterio mínimo permitirá afirmar que el piloto fue exitoso?
